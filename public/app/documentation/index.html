Systemet er opbygget på tre hoved komponeneter, disse er følgende: Frontend, som er primært er lavet i Angular. Backend, hvor vi bruger Nodejs som også er vores API. Database, Vi bruger MongoDB (Mongolab) som vores database. Disse teknologier er sat sammen så de gør brug af hinanden. Vores mest centrale ting i projektet er vores database, som gemmer på alle wiki siderne og deres information. Vores API er det eneste som har direkte adgang til vores database (gennem Facaden). Frontend får alt informationen gennem API'et fra databasen.

I denne omgang har vi ikke været helt så grundige i vores backend-tests, som i tidligere projekter. Vi har lidt følt at fokus burde være på frontend testing i dette projekt. I frontend testene er backend og REST-API kald mocked væk. Dog har vi testet backend, med vailide forspørgsler. Altså på titler, kategorier osv. som vi ved findes i databasen. Derfor er det store hul i testene hvordan backend håndterer forespørgsler der ikke findes. Eksempelvis bliver backenden ikke testes i hvorledes den reagerer på en søgestreng der ikke findes.
